## What You Get For USD 20

- Ranked teardown of the main conversion problems

## Why The Flow Is Structured This Way

- The first reply is a short free preview so fit is visible before payment.
- The paid scope stays narrow so delivery can stay fast and consistent.
- Everything is AI-assisted and text-only, which keeps the service honest and lightweight.

## How To Request One

1. Open a GitHub issue using the `Landing Page Teardown Request` template.
2. Include the landing page URL, what the page sells, the page goal, intended audience, and the main concern.
3. Receive a short public preview in the issue.
4. If the preview looks useful, pay USD 20 by PayPal for the full teardown.
5. Receive the final teardown within 24 hours after payment confirmation.

## Before You Open An Issue

- Keep the request to one public page.
- State the main action you want the page to drive.
- Say what audience the page is trying to convert.
- Add one constraint you want weighted heavily in the review.
- Do not post private URLs, internal metrics, or confidential launch details.

## Ground Rules

- The work is AI-assisted.
- No promises are made about conversion lifts or sales outcomes.
- No site access, account access, or credentials are required.
- GitHub issue intake is public, so do not post confidential details or private URLs.
