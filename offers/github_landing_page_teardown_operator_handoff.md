# Publish This Bundle

Use this file from inside `dist/landing-page-teardown-pack/` during the one required GitHub publication step.

## Create The Repo

- Repository name: `landing-page-teardown-pack`
- Visibility: `Public`
- Description: `USD 20 AI-assisted landing page teardown pack for founders and small software products.`

## Upload This Folder

Upload the full contents of this folder to the new repository root, including:

- `README.md`
- `docs/`
- `.github/`
- `offers/`

If GitHub asks whether to replace the default `README`, use the files from this folder.

## Enable Pages

- Open repository `Settings`
- Open `Pages`
- Source: `Deploy from a branch`
- Branch: `main`
- Folder: `/docs`
- Save

## Set The Repo Website Link

- Wait for the Pages URL to appear in GitHub
- Open the repo main page
- In the right-side `About` box, edit `Website`
- Paste the live GitHub Pages URL
- Save so GitHub visitors can click straight to the rendered offer page

## Check Before Leaving

1. Open the public repo and confirm the README reads clearly on GitHub without relying on raw HTML links.
2. Confirm the repo `About` website link points to the live GitHub Pages URL.
3. Open the GitHub Pages URL and confirm the offer page loads.
4. Click `Open request issue` on the Pages site and confirm it opens the repo issue chooser.
5. Confirm the `Landing Page Teardown Request` template appears and blank issues are disabled.
6. Confirm the GitHub Pages sales page links to a live `before-you-pay.html` objection-handling page.
7. Confirm the GitHub Pages sales page links to a live `examples.html` rewrite-examples page, not a missing page.
8. Confirm `https://<pages-url>/og-preview.svg` loads so the shared social preview asset is live.

## Send Back These Two URLs

- Repository URL
- GitHub Pages URL

## Return Message Template

Use `PUBLICATION_RECEIPT.md` in this bundle and replace the placeholders before sending the result back.

Success means both URLs are live, the repo `About` website link points to Pages, the Pages CTA opens the issue chooser correctly, the pre-purchase objections page loads, the rewrite-examples proof page loads, and the shared social preview asset loads.
