# GitHub Landing Page Teardown Payment Handoff

Use this file after a prospect has received a useful preview and is deciding whether to buy the
full USD 20 pack.

## 1. Standard Payment Request

Use when the preview landed well and the buyer wants to continue.

```md
If you want the full teardown, the next step is USD 20 paid upfront by PayPal.

After payment is confirmed, delivery target is within 24 hours in Markdown.

To continue, send payment to: `[PAYPAL ADDRESS OR LINK]`

Then reply here with:
- confirmation that payment was sent
- the exact public page URL to review
- one concern you want weighted most heavily

What the paid pack includes:
- ranked conversion problems
- three replacement headlines
- one CTA rewrite
- the first change to ship
```

## 2. Short Reminder Reply

Use if the prospect says they will pay but then goes quiet.

```md
The full teardown is still available if you want it.

Current path is the same: USD 20 paid upfront by PayPal, then delivery target is within 24 hours
after payment confirmation.

If you want to continue, send payment to `[PAYPAL ADDRESS OR LINK]` and reply with confirmation
plus the exact page URL.
```

## 3. Payment Confirmed Reply

```md
Payment confirmed.

I have the page URL and I am preparing the full teardown now.

Delivery target: within 24 hours in Markdown.
```

## 4. Internal Notes

- Keep the public issue truthful: one page, one paid pack, no hidden upsell.
- Reuse the same language across the issue, Pages site, and delivery flow.
- If the buyer needs privacy or a non-public URL, decline instead of stretching scope.
